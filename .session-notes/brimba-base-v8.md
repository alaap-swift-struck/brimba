## Key decisions this session (and why)
- **Sonnet 5 @ low effort, not Haiku.** Owner asked about Haiku; declined — the agent does multi-step tool orchestration (Haiku is weakest there), Haiku doesn't support the `effort` knob we use, and Sonnet 5 at low effort is already ~free per turn (0 thinking tokens). Cost levers are effort + the daily cap, not a weaker brain.
- **Confirm "only destructive asks."** Owner chose the lowest-friction option knowingly; the server still enforces rights on every action, so "no prompt" ≠ "no guardrail."
- **Screen-tracing = "drive the real screens" (LIGHT), not a fake cursor.** It navigates to the actual screen + opens the actual dialog + highlights — which genuinely teaches the manual path; a simulated cursor teaches a mock and is fragile.
- **Two perf wins DEFERRED, on purpose.** The permission+data D1 batch (`d1QueryMulti`) touches the security gate (deny-before-read); the learning-list `body` trim touches the list-doubles-as-detail-source design. Both medium-risk → documented in EDGE-CASES.md for the owner to review rather than shipped autonomously.
- **Activity sort fixed host-only with a display trade-off** (see Open questions) — because the proper fix needs a library change and the owner was away.
- **XSS fix via `escapeAttr` + a pure-`.ts` extraction** — the renderer was split to `web/lib/agent-markdown-html.ts` because importing the `.tsx` into a test tripped the bundler; the `.ts` is unit-testable.

## Established patterns / conventions (reinforced this session)
- **Every build phase:** an investigation Workflow (root-cause, file:line) → a file-partitioned build Workflow (agents own non-overlapping files, build to a locked contract) → integrate `npm run check` → deploy:staging → smoke → (validate) → commit. Contract-first parallel builds integrated with zero rework (e.g. the streaming SSE contract).
- **Library-is-lego:** never fork `@swift-struck/ui`; hand the owner a ready prompt for library changes; they apply + republish + we re-install.
- **The Laws of the Base** are machine-checked (RULES.md ⇄ `shared/rules/registry.ts` ⇄ tests read off disk). Adding a law requires all three (R8 added this session). Break a law → red build.
- **Bulk pattern:** one gated endpoint fans out server-side (ids validated + capped), `publishChange` per changed row (never refetch), confirm-with-count.
- **Streaming on Workers:** `TransformStream` response, `Accept: text/event-stream`, `X-Accel-Buffering: no`, one terminal event, gateway service-binding streams through unbuffered.

The demo uses the DEFAULT `surface="card"` → List draws its OWN rounded, `overflow-hidden` Card → the full-bleed selected/hover bg (`bg-primary/10`) clips to the rounded corners. The mismatch surfaced with library **0.3.0**, which flipped the engine's `renderList` to pass `surface="none"`. Engine renders `<List surface={recipe.surface}>` (screen-renderer.tsx:571) with NO className seam → can't round from the recipe; the list root always carries `divide-y overflow-hidden` (list.tsx:73).
